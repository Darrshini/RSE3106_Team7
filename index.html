Designed for Accessibility

Built With the User in Mind

The user never needs to look at a screen. A tap anywhere is enough — audio and haptic feedback handle the rest.

Tap Anywhere to Start

One full-screen tap zone — no buttons to find. Audio confirms activation.

State-Managed Input

A 13-state FSM ignores accidental touches so only intentional taps register.

Audio-First Guidance

Crossing status and directions are spoken aloud — the screen is for caregivers only.

Adjustable Feedback

Haptic intensity, speech rate, and volume are tuneable by a caregiver.

Future direction: Our current web app still requires a browser launch, which isn't independently accessible. We see T.O.P.H as a proof of concept for the core navigation intelligence. The next step is a native app activated entirely by voice command — no screen navigation or typing required to start a session.

About Us

Our Story

T.O.P.H began as a Systems Engineering project aimed at a genuine gap in assistive technology: while tools like white canes and guide dogs help visually impaired pedestrians avoid obstacles, few solutions actively help them judge whether it is safe to cross a road.

The result is a two-part system — a wearable glasses unit with a live camera feed, and a browser-based web app on the user’s phone, communicating over a secure WebSocket relay.

How T.O.P.H works, what hardware it uses, and why we built it this way.

A camera on the glasses unit streams live video to an onboard YOLO11 segmentation model, which identifies crossing infrastructure and pedestrian signals in real time. A double-confirmation safety gate then checks the crossing state before any "safe to cross" signal is sent to the user.
We started with an ESP32 camera but it failed all three tests: images pixelated at 10 metres, motion caused total blur, and thermal throttling dropped video from 30fps to under 1fps within minutes. We upgraded to a Raspberry Pi Zero 2W with a Camera Module, boosting AI detection confidence from ~70% to ~94%.
Through the phone-based web app, which communicates with the glasses unit over a secure WebSocket relay. Feedback is delivered as audio cues via the Web Speech API, alongside simple tap and hold gestures for the user to confirm actions.
We started as a native Android app, but every code change required a full compile-sign-deploy cycle. With less than two months to build both hardware and software, we switched to a browser-based web app — instant push-and-test, cross-device compatibility, and standard web APIs for everything.
A stall-detection timer in the crossing flow recognizes when a user has paused, rather than assuming continuous motion, and adjusts its guidance accordingly instead of issuing outdated instructions.
Pedestrian crossing infrastructure is resolved through the Overpass API querying OpenStreetMap, giving reliable, cost-free access to crossing locations without relying on a paid mapping service.
